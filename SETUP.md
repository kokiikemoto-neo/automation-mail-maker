# Writing Robot セットアップ手順（VS Code）

このプロジェクトを VS Code で動かし、ロボットで直線を1本書かせるまでの手順です。
上から順に進めてください。各ステップの「✅ 確認」が通ったら次へ進みます。

---

## 前提

- GitHub リポジトリとフォルダは作成済み
- Python は python.org からインストール済み
- ロボットはペン・紙セット済み、USB ケーブルあり

---

## ステップ 0：VS Code でフォルダを開く

1. VS Code を起動
2. メニュー「ファイル」→「フォルダーを開く」
3. 作成した `writing-robot` フォルダを選ぶ

✅ 確認：左側のエクスプローラーに `writing-robot` の中身（GitHub から作られた `.git` など）が見える

---

## ステップ 1：Python 拡張機能を入れる

1. 左端の四角が4つのアイコン（拡張機能）をクリック
2. 検索欄に `Python` と入力
3. Microsoft 製の「Python」をインストール

✅ 確認：インストール後、右下に Python のバージョンが表示される

---

## ステップ 2：ターミナルを開く

1. メニュー「ターミナル」→「新しいターミナル」
2. 画面下にターミナルが開く

✅ 確認：プロンプトの先頭が `writing-robot` フォルダのパスになっている

もしターミナルが PowerShell で、後でコマンドが通らない場合は、
ターミナル右上の「＋」の横の `∨` から「Command Prompt」を選び直すと素直に動きます。

---

## ステップ 3：Python が動くか確認

ターミナルに次を打って Enter：

```
python --version
```

✅ 確認：`Python 3.x.x` と表示される

> もし `python` で何も出ない／ストアが開く場合は、代わりに `py --version` を試してください。
> `py` で動いた場合は、以降のコマンドの `python` を `py` に読み替えてください。

---

## ステップ 4：必要なライブラリを入れる

ロボットと通信するための `pyserial` を入れます：

```
pip install pyserial
```

✅ 確認：最後に `Successfully installed pyserial-x.x` と出る
（`already satisfied` と出た場合は既に入っているので問題なし）

---

## ステップ 5：ロボットの COM 番号を確認

1. ロボットの電源を入れる（赤いスイッチ ON）
2. USB ケーブルを PC に挿す
3. キーボードで `Windows キー + X` →「デバイスマネージャー」を開く
4. 「ポート (COM と LPT)」を展開
5. `USB-SERIAL CH340 (COM3)` のような表示を確認し、COM の番号を控える

✅ 確認：COM 番号がわかった（前回は COM3 だった）

> 番号が COM3 以外だった場合は、次のステップで作るファイルの `SERIAL_PORT` を
> その番号に書き換えます。

---

## ステップ 6：テスト用スクリプトを置く

1. エクスプローラーで `writing-robot` フォルダを右クリック →「新しいファイル」
2. ファイル名を `test_write.py` にする
3. 別途渡している `test_write.py` の中身をすべてコピーして貼り付け
4. 保存（`Ctrl + S`）

✅ 確認：ファイルの先頭に COM 番号の設定（`SERIAL_PORT = 'COM3'`）が見える

> COM 番号が COM3 以外なら、ここで `'COM3'` を実際の番号に書き換えて保存。

---

## ステップ 7：直線テストを走らせる

ロボットの電源が入っていて、USB が挿さっている状態で、
ターミナルに次を打って Enter：

```
python test_write.py 1
```

▶ 横線1本と縦線1本を、紙の左下あたりに書きます。

⚠️ 最初の1回は、手をロボットの電源スイッチの近くに置いてください。
   変な方向に動いたら、すぐ電源を切れるように。

✅ 確認：紙に横線と縦線が書かれた

---

## ステップ 8：結果をチェック

書かれた線について、次の3点を見てください：

| 見るところ | OK | NG の場合 |
|---|---|---|
| かすれ | くっきり書けている | ペンの高さ・圧の調整が必要 |
| まっすぐさ | 直線になっている | 機械の精度・速度の問題 |
| 速度 | 違和感ない速さ | `FEED` の数値を調整 |

- 速度を変えたい：`test_write.py` 冒頭の `FEED = 1000` の数字を変更
  （小さくすると遅く・きれいに、大きくすると速く）
- ペンが下りない／上がらない／紙に着きっぱなし：
  ペン制御の設定（`PEN_DOWN` / `PEN_UP`）の調整が必要 → 症状をメモしておく

---

## うまくいったら

ステップ1（直線）が通ったら、次の2つを順に試します：

```
python test_write.py 2    # 四角形（角がきちんと閉じるか）
python test_write.py 3    # ひらがな「し」（曲線がなめらかか）
```

ここまで通れば、フォントを使った宛名書きへ進む土台が完成です。

---

## トラブル時に伝えてほしいこと

うまくいかなかったときは、次を教えてもらえると原因を切り分けられます：

1. どのステップで止まったか
2. ターミナルに出たメッセージ（赤い文字があればそれも）
3. ロボットの動き（全く動かない／動くが書かない／変な方向に動く 等）
