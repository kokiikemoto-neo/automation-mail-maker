# 便箋作成ウェブアプリ（GAS版）— Phase 6 案B

誰でもブラウザから会社を選び、Gemini が文面を下書きし、「便箋作成」を押すと
ロボットPCの**順番待ち（キュー）**に入る。ロボットPCはキューを見に行って手書きする。

```
ブラウザ（誰でも）            Googleスプレッドシート          ロボットPC（Python常駐）
GASウェブアプリ                （Orders シート＝キュー）
 会社選択                          │                            数秒ごとに見に行く
 「文面を生成」─ Gemini API ─▶ 下書き                          「待ち」を見つけたら
 「便箋作成」 ───書込─────▶  status:待ち  ◀────確認──────  処理中→Gコード→手書き
                                  status:完了 ◀───更新───────  書き終わったら完了
```

## 構成ファイル

| ファイル | 役割 |
|---|---|
| `Code.gs` | サーバー側。画面配信・Gemini文面生成・キュー書込・状況取得 |
| `index.html` | 画面（会社入力・文面生成・便箋作成ボタン・状況一覧） |
| `appsscript.json` | マニフェスト（ウェブアプリ公開設定・必要な権限） |

## セットアップ（初回だけ）

### 1. スプレッドシートを用意
1. Google ドライブで新規スプレッドシートを作る（名前は任意）
2. 拡張機能 → Apps Script を開く（これで「コンテナバインド」になり `SHEET_ID` は空のままでOK）

### 2. コードを貼る
1. Apps Script エディタで、`Code.gs` の中身を貼る
2. ファイル＋ → HTML →名前を `index` にして `index.html` の中身を貼る
3. プロジェクトの設定（歯車）→「`appsscript.json` マニフェストをエディタで表示」にチェック →
   `appsscript.json` の中身を貼る

### 3. Gemini API キーを登録（画面には出ない安全な場所）
1. [aistudio.google.com/apikey](https://aistudio.google.com/apikey) でキー（`AIza...`）を取得
2. プロジェクトの設定 → スクリプト プロパティ → プロパティを追加
3. 名前 `GEMINI_API_KEY` / 値 `AIza...`

### 4. 初期化と権限承認
1. エディタの関数選択で `setup` を実行 → 権限の承認ダイアログを許可（Orders シートが作られる）
2. `generateMessage_test` を実行 → ログに文面が出れば Gemini 連携OK

### 5. ウェブアプリとして公開
1. デプロイ → 新しいデプロイ → 種類「ウェブアプリ」
2. 実行するユーザー：**自分** ／ アクセスできるユーザー：**全員**
3. 発行された URL を5人に共有

> 「全員」で公開しても、APIキーはサーバー側（スクリプト プロパティ）にあるので画面からは見えない。

### （任意）会社マスタ
`Companies` という名前のシートを作り、A列に会社名を並べると、画面の会社名入力に候補が出る。

## Orders シートの列（ロボットPC側との「契約」）

ロボットPC側のプログラムは、この形を前提に読む。**列の順番は変えない。**

| 列 | 項目 | 例 | 説明 |
|---|---|---|---|
| A | ID | `ORD-20260602-141530-742` | 注文の一意キー |
| B | 受付時刻 | 2026/06/02 14:15 | 作成日時 |
| C | 担当者 | 池本 | 誰が仕込んだか |
| D | 会社名 | 株式会社サンプル | |
| E | 宛名 | 採用ご担当者 | 空でも可（会社単独なら御中など） |
| F | 文面 | （手書きする本文） | |
| G | ステータス | 待ち / 処理中 / 完了 / エラー | ロボットPCが更新する |
| H | 処理時刻 | 2026/06/02 14:20 | 書き終わった時刻 |
| I | 備考 | 通信切れで再試行 等 | エラー内容など |

### ステータスの流れ
```
待ち → （ロボットPCが拾う）→ 処理中 → （手書き完了）→ 完了
                                      └（失敗）→ エラー（備考に理由）
```

## 次のステップ：ロボットPC側の常駐プログラム（ポーリング）

このウェブアプリは「キューに積む」までが担当。実際に書くのは別途、ロボットPCで動く
Python 常駐プログラムが行う（次フェーズ）。やることの概要：

1. Google Sheets API でこの Orders シートに接続（サービスアカウント or OAuth）
2. 数秒ごとに「ステータス＝待ち」の行を1件取得 → 「処理中」に更新
3. その行の宛名・文面を、既存の `kanji2gcode.py` / `batch_write.py` に渡して手書き
4. 書き終わったら「完了」、失敗したら「エラー」＋備考に理由を記録
5. 1台のロボットしかないので、同時処理はせず1件ずつ順番に

> このプログラムを作るときに声をかけてください。既存の Python 資産にポーリングの
> 外殻をかぶせるだけなので、土台はできています。
