# ロボットPC セットアップ手順（常駐プログラム）

スプレッドシートの「待ち」注文を、ロボットPCが自動で手書きする常駐プログラム
（`robot_daemon.py`）を動かすまでの手順。**ロボットがUSB接続されているPCで**行う。

```
GASウェブアプリ ──書込──▶ スプレッドシート（Orders）◀──確認── robot_daemon.py（このPC）
  「便箋作成」              status: 待ち                    5秒ごとに見に行く
                            status: 完了 ◀──更新────────   手書きして完了にする
```

---

## A. サービスアカウント（機械用アカウント）を作る ※初回だけ

ロボットPCがブラウザ無しでスプレッドシートを読み書きするための「機械用アカウント」を作る。
これは個人アカウントではなく、Google Cloud プロジェクト（＝会社の持ち物にできる）に属する。

### A-1. Google Cloud プロジェクトを用意
1. [console.cloud.google.com](https://console.cloud.google.com) を会社アカウントで開く
2. 上部のプロジェクト選択 →「新しいプロジェクト」→ 名前例 `binsen-robot` → 作成
3. 作ったプロジェクトを選んだ状態にする

### A-2. 必要なAPIを有効化
1. 左メニュー「APIとサービス」→「ライブラリ」
2. **Google Sheets API** を検索 → 有効にする
3. （名前でシートを開く場合のみ）**Google Drive API** も有効にする

### A-3. サービスアカウントを作成
1. 「APIとサービス」→「認証情報」→「認証情報を作成」→「サービスアカウント」
2. 名前例 `robot-daemon` → 作成して続行 → 役割は不要（そのまま完了）
3. できたサービスアカウントの**メールアドレス**を控える
   （`robot-daemon@binsen-robot.iam.gserviceaccount.com` のような形）

### A-4. 鍵（JSON）をダウンロード
1. そのサービスアカウントをクリック →「キー」タブ →「鍵を追加」→「新しい鍵を作成」
2. 形式 **JSON** → 作成 → JSONファイルがダウンロードされる
3. このファイルを `robot_daemon.py` と同じフォルダに置き、名前を **`service_account.json`** にする

> ⚠️ この鍵は**パスワードと同じ**。GitHub には絶対に上げない（`.gitignore` 済み）。

### A-5. スプレッドシートを機械用アカウントに共有
1. 注文キューのスプレッドシート（**便箋作成キュー（自動生成）**）をブラウザで開く
2. 右上「共有」→ A-3で控えた**サービスアカウントのメール**を入力
3. 権限を**編集者**にして共有（通知は不要）

これで、機械用アカウントがこのシートだけを読み書きできる状態になる。

## B. PC側の準備

### B-1. ライブラリを入れる
ロボットPCのターミナル（このフォルダ内）で：
```
pip install -r requirements-daemon.txt
```

### B-2. 設定を自分の環境に合わせる
`robot_daemon.py` の上部 `SETTINGS` を編集：

| 項目 | 説明 |
|---|---|
| `SPREADSHEET_ID` | スプレッドシートURLの `/d/` と `/edit` の間の文字列。空なら名前で開く |
| `SERIAL_PORT` | ロボットのCOMポート（デバイスマネージャーで確認。例 `COM3`） |
| `PAPER_W` / `PAPER_H` | 便箋の幅・高さ(mm) |
| `ORIGIN_X` / `ORIGIN_Y` | 書き出し原点(mm) |
| `CHAR_HEIGHT` | 文字高さ(mm) |
| `VERTICAL` | 縦書きなら True |

> `SPREADSHEET_ID` を入れるのが確実（名前開きは Drive API と表記ゆれに弱い）。

## C. 動かす

### C-1. まず空運転で流れを確認（ロボットは動かさない）
```
python robot_daemon.py --dry
```
- 「キューに接続しました」と出ればサービスアカウント連携OK
- ウェブアプリから「便箋作成」すると、数秒後にその注文を拾い、範囲チェックして
  「完了」になる（実機は動かない）。スプレッドシートのステータス変化で確認できる。

### C-2. 本番（ロボットで実際に書く）
ロボットの電源を入れ、紙をセットして：
```
python robot_daemon.py
```
- 「待ち」を見つけるたびに1件ずつ手書きし、「完了」に更新する
- 停止は **Ctrl+C**

> ⚠️ 最初の1回は電源スイッチの近くに手を置き、変な動きをしたらすぐ切れるように。

## 状態の意味（スプレッドシートの G列）

| ステータス | 意味 |
|---|---|
| 待ち | ウェブアプリが積んだ直後。これから処理される |
| 処理中 | このプログラムが拾って手書き中 |
| 完了 | 手書き成功（H列に時刻） |
| エラー | 失敗（I列の備考に理由。はみ出し・通信切れ等） |

## よくあるトラブル

- **「鍵ファイルが見つかりません」** → `service_account.json` が同じフォルダにあるか確認
- **接続で権限エラー** → A-5 のシート共有（サービスアカウントのメール）を忘れていないか
- **はみ出しエラー** → `CHAR_HEIGHT` を小さく、または `PAPER_W/H`・`ORIGIN` を見直す
- **COMポートで失敗** → デバイスマネージャーで番号を確認し `SERIAL_PORT` を修正
- **筆画データが無い文字** → その文字は飛ばして書く。`fonts/kanjivg/` に追加すれば対応
